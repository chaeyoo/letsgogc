> 자소서 3번(직무역량)·1차 직무면접의 뼈대. 공고 요구역량을 **내 실제 경험**에 매핑하고, 부족분을 학습·프로젝트로 메우는 계획.

## 1. FDE(Forward Deployed Engineer)란

Palantir가 만든 직무 모델. GC녹십자가 AX 조직에 이 개념을 도입.

- **현장 밀착형 엔지니어** — 고객(=사내 현업 부서)에 직접 투입돼 함께 일한다.
- **"슬라이드가 아니라 코드를 배포한다"** — 실제 작동하는 솔루션을 빠르게 프로토타이핑·전달.
- **End-to-end 오너십** — 문제 정의 → 개발 → 배포까지 소규모 팀으로 전담(스타트업 CTO처럼).
- **비즈니스↔기술 통역사** — 비개발 현업과 기술팀 사이 소통이 필수 역량.
- **성과(outcome)로 측정** — 투입 시간이 아니라 실제 업무 혁신으로 평가.

> **GC 맥락:** R&D·제조·경영지원 등 사내 부서에 들어가, 반복업무·데이터 문제를 **Agentic AI 솔루션(Hey.GC 2.0 생태계, MCP 구조)** 으로 직접 풀어내는 사람.

### 💡 왜 내가 FDE에 맞나 (한 문장)
> **"비개발 현업이 쓸 도구를 직접 만들어 온 풀스택 개발자 + 5년 내내 헬스케어 도메인 + 이미 AI 에이전트(Claude Code)로 실제 서비스를 배포해 본 사람."** — FDE의 3요소(현장밀착·풀스택·outcome)를 이미 하고 있었다.

## 2. 공고 원문 핵심 (2026)

**주요 업무**
- 생성형 AI 서비스 개발 — **RAG 최적화 · Agentic Workflow · Function Calling · Subagent · MCP**
- **AI 기반 개발도구(Claude Code, Cursor 등)** 로 FE·BE 개발
- **비즈니스 워크플로우 분석** → 최적 프로세스 설계

**자격요건 [필수]**
- Python, JavaScript, Dart, Rust, Golang 등으로 개발 가능
- **FastAPI, FastMCP, PydanticAI, LangChain, LangGraph, Enterprise LLM API** 활용 경험
- **ML/DL 이론 이해** 및 논리 기반 모델 설계
- 사용자 입장에서 **"왜"** 를 던지고 **구조적 해결**을 설계

**자격요건 [우대]**
- 인공지능·컴퓨터공학 전공
- **제약/바이오/헬스케어 산업 이해** ← 나의 강점 (생명과학 전공 + 헬스케어 커리어)
- **사내 업무 프로세스에 AI를 이식해 생산성을 개선한 실무 경험** ← FDE 정체성 = 자소서 3번 핵심

## 3. 요구역량 ↔ 내 경험 매핑표 ⭐ (자소서 3번·면접의 핵심)

> 수준 표기: 🟢 강점(경험으로 증명) · 🟡 인접·부분경험(연결 가능) · 🔴 미보유(프로젝트로 보완)

| 공고 요구 | 핵심 개념 | 내 현재 수준 | 근거 / 보완계획 |
|---|---|:---:|---|
| **AI 개발도구(Claude Code)** | 에이전트로 실제 개발 | 🟢 | **KB헬스케어 예약 사이트 FE를 Claude Code 에이전트로 단독·2개월 완료.** 공고 명시 도구를 이미 실무에 적용 — 최고의 어필 소재 |
| **풀스택(FE·BE)** | React FE + API·DB BE | 🟢 | 5년 풀스택. 마크로젠 LIMS / 에임메드 앱 BE·어드민 / KB FE. 헬스케어 도메인 일관성 |
| **제약·바이오 도메인** | 산업 이해(우대) | 🟢 | **생명과학 전공 + 마크로젠(유전자분석)·에임메드·KB헬스케어** 커리어 전체가 헬스케어 |
| **생성형 AI 서비스** | 챗봇·LLM 앱 | 🟡 | 에임메드 **챗봇 빌더 시스템** 풀스택 개발(생성형 AI 인접). LLM API 연동 경험으로 확장 |
| **비즈니스 워크플로우 분석** | 현업 문제→구조 설계 | 🟢 | 챗봇 빌더·건강관리 어드민 등 **비개발 현업이 쓰는 도구를 설계**해 옴 = FDE 통역사 역량 |
| **Function Calling** | 구조화 출력·툴 콜링 | 🟡 | 개념 이해(용어사전 9장). 프로젝트에서 직접 구현해 🟢로 |
| **RAG 최적화** | 청킹·임베딩·리랭킹·평가 | 🔴→🟡 | 이론 정리 완료(용어사전 8장). **프로젝트에서 미니 RAG 파이프라인 구축**으로 증명 |
| **Agentic Workflow / Subagent** | 에이전트 루프·오케스트레이션 | 🔴→🟡 | 이론 정리(9장) + Claude Code 사용자로 에이전트 UX 체감. 프로젝트로 직접 구현 |
| **MCP / FastMCP** ⭐GC핵심 | MCP 서버·사내연동 | 🔴→🟡 | **최우선 보완.** FastMCP로 사내도구 흉내낸 MCP 서버 직접 제작(프로젝트) |
| **LangGraph** | 그래프 상태 에이전트 | 🔴→🟡 | 튜토리얼 1개 완주 + 프로젝트 에이전트 흐름에 적용 |
| **PydanticAI** | 타입안전 LLM 앱 | 🟡 | Pydantic/타입 기반 백엔드 경험 있음 → 학습곡선 낮음. 프로젝트에 적용 |
| **FastAPI** | 파이썬 API 백엔드 | 🟡 | BE 경험 보유, FastAPI로 프로젝트 백엔드 구현해 🟢로 |
| **ML/DL 이론** | 모델 설계 논리 | 🟡 | 용어사전 2~5장으로 "설명 가능" 수준 확보. 면접 대비 |
| **다양한 언어(Rust/Go/Dart 등)** | 폭넓은 개발 | 🟡 | JS/Python 강점 + Dart(Flutter) 접점. "필요시 습득" 학습력 어필 |

## 4. 학습·보완 계획 (부족분 → 프로젝트로 통합)

> 흩어서 공부하지 말고 **계획서 3장 프로젝트 하나에 다 녹인다.** ("사내 문서 RAG + MCP 툴 연결 Agentic 어시스턴트")

| 우선 | 항목 | 학습 방법 | 프로젝트 연결점 | 상태 |
|:---:|---|---|---|:---:|
| 1 | **MCP + FastMCP** | 공식 문서 + FastMCP로 도구 2~3개 노출 | 일정/문서검색 툴을 MCP 서버로 | ⬜ |
| 2 | **RAG 미니 파이프라인** | 청킹→임베딩→검색→**리랭킹**→평가 | 사내문서(샘플) 검색·요약 | ⬜ |
| 3 | **Agentic + Function Calling** | 에이전트 루프 + 툴 콜링 | 에이전트가 RAG·MCP툴 자율 호출 | ⬜ |
| 4 | **LangGraph or PydanticAI** | 튜토리얼 1개 완주 | 에이전트 흐름 제어(택1) | ⬜ |
| 5 | **FastAPI 백엔드 + 웹 챗 UI** | 기존 풀스택 역량 활용 | 데모를 실제 작동하게 | ⬜ |
| — | **ML/DL 이론** | 용어사전 2~5장 소리내어 설명 연습 | 면접 대비(코드 아님) | 🟡 진행 |

> 💊 **제약사에서 AI가 푸는 실제 문제 3선**(면접·프로젝트 시나리오용): ① 규제/SOP 문서 검색·요약(RAG) ② 실험·검진 데이터 정리·보고서 자동화 ③ 사내 반복업무(예약·일정·문의) 에이전트 자동화. → 내 KB헬스케어(검진예약)·에임메드(보고서생성) 경험과 직접 연결됨.
