### S (상황)
지원 직무(FDE)와 GC의 방향(`Hey.GC 2.0` = Agentic AI + MCP)에 정확히 겹치는 역량을
증명하기 위해, 제약 **RA(규제업무)·PV(약물감시) 담당자의 실제 반복업무**(규정 검색·기한 관리·
이상사례 케이스 처리)를 푸는 AI 어시스턴트를 직접 만들기로 했다.

### T (과제)
공고 필수 스택(**RAG 최적화 · MCP/FastMCP · Agentic Workflow · Function Calling · FastAPI**)을
한 프로젝트에서 **작동하는 형태로** 구현하되, 규제 산업이 요구하는 **근거 추적성**과
우대 요건인 **제약/바이오 산업 이해**(RA·PV 도메인 워크플로)까지 갖출 것.

### A (행동)
- **RAG 파이프라인**을 직접 설계: 구조 청킹+overlap → 하이브리드 검색(벡터+BM25) → **리랭킹**.
  무거운 ML 라이브러리 없이 순수 파이썬으로 TF-IDF/BM25/코사인/리랭커를 구현해 **어디서든 실행** 가능하게 함.
- **FastMCP로 MCP 서버**를 만들어 RA·PV 업무(문서검색·마감일·체크리스트·이상사례 처리)를 도구로 노출.
  모델과 도구를 MCP로 분리해 `Hey.GC 2.0`과 같은 확장 구조를 구현.
- **PV 케이스 처리 워크플로**(트리아지→인과성(WHO-UMC) 제안→용어 코딩→ICSR 보고서 초안)를
  규칙 기반(결정론)으로 구현 — 컴플라이언스 계산은 LLM이 아닌 감사 가능한 도구가 담당.
  환자 **개인정보(PII)는 에이전트 입구+MCP 도구 계층 2겹으로 마스킹**(도구 단독 사용 경로도
  재마스킹)해 외부 API로 나가지 않게 함.
- **에이전트**가 MCP 도구를 자율 호출(Function Calling)하도록 하고, LLM 키가 없어도
  근거 기반으로 답하는 **graceful degradation**을 넣어 데모 무중단성을 확보.
- **FastAPI + 웹 챗 UI**로 풀스택 완성, 답변에 **출처·도구 트레이스**를 노출.
- **RAG 평가 스크립트**(Hit@1·MRR·ContextRecall)로 벡터 단독 대비 하이브리드의 개선을 수치로 검증.

### R (결과)
- 데이터→RAG→MCP→에이전트→API→UI→평가까지 **end-to-end 작동 데모** 완성.
- 검색 평가에서 하이브리드가 벡터 단독 대비 **ContextRecall 0.781 → 0.844** 개선을 확인
  (리랭킹·질의확장까지 적용 시 **0.969**, Hit@1 0.875 → **1.000** — 32문항 기준.
  32문항은 튜닝에 쓴 개발셋 — 홀드아웃 6문항은 별도 유지, 라벨 교정 후 1.000).
- PV 워크플로도 라벨 22케이스로 수치 검증 — 중대성·기한·인과성·보고요건 판정 **1.000**,
  코딩 정밀도 **1.000**(무관용)에 재현율 갭(0.792)은 확장 지점으로 정직하게 남김.
- ⬜ (추가로 채울 것: 스스로 느낀 임팩트, 예: "규정 확인에 걸리던 시간을 체감상 크게 단축")

### 배운 점 / 느낀 점
- **RAG는 '임베딩 넣고 검색'이 아니라 최적화 단계(청킹·하이브리드·리랭킹·평가)의 문제**임을 체득.
- MCP로 도구를 분리하면 **한 번 만든 도구를 여러 에이전트가 재사용**하는 확장성이 생긴다는 걸 코드로 이해.
- 제약 규제 산업에선 **정확도보다 출처 추적·환각 억제**가 먼저라는 감각.
- 컴플라이언스 업무(보고기한·요건 판정)는 **LLM의 확률적 추론이 아니라 결정론적 도구**에
  맡겨야 감사 가능하다는 것 — LLM(이해·조합)과 규칙(판정·계산)의 경계 설계.
- ⬜ (본인 언어로: FDE로서 현업 문제를 코드로 푸는 방식이 왜 맞는지)
