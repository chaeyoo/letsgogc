# 06. 경험 인벤토리 — 자소서 2·3번 재료 워크시트

> **목적:** 자소서 2번(중도포기 위기 극복)·3번(직무역량 향상, 2개↑)에 쓸 경험을 발굴·정리한다.
> ⬜ 는 본인이 추가로 채우면 좋은 칸.

---

## 👤 지원자 프로필 요약
- **전공:** 생명과학 (⭐ 제약/바이오/헬스케어 산업 이해 = 공고 우대사항 직격)
- **총 경력:** 웹개발 5년 (대부분 풀스택)
- **커리어 라인이 전부 헬스케어 도메인:**
  - **마크로젠** (유전자 분석) — 실험실 정보관리 시스템(LIMS) 풀스택, 레거시 Java
  - **에임메드** (디지털 헬스케어) — 챗봇 빌더 시스템 / 임직원 건강관리 앱 백엔드 / 임직원 건강관리 어드민 풀스택 / 직무스트레스 검사 보고서 생성 사이트 풀스택
  - **KB헬스케어** (現, 1년 1개월) — 건강검진 예약 어드민 + 사용자 사이트 FE

> 💡 **핵심 어필 포인트:** ① 헬스케어 도메인 일관성 ② 풀스택 ③ 생성형 AI 인접경험(챗봇 빌더) ④ **AI 개발도구(Claude Code) 실전 활용** — 4가지가 공고와 정확히 맞물림.

## STEP 3. 문항별 소재 배정 (✅ 확정 — 세 문항 서로 다른 프로젝트)

### 📝 문항 2 — 중도 포기할 수 있었지만 완수한 경험 — ✅ KB 건강검진 리뉴얼
> **위기:** 사업파트가 협의 안 된 디자인 시안으로 영업 → 확정 안 된 화면으로 기존+신규 기능을 2달 내 개발
> **극복:** 기획자·사업부와 협의하며 기획 병행(비즈니스 워크플로우 분석→프로세스 설계) + Claude Code(컨텍스트 엔지니어링)로 개발 가속
> **직무연결:** 자격요건'비즈니스 워크플로우 분석·프로세스 설계' + 주요업무'AI 개발도구 활용 개발' / 가치: 창의도전
- **① 맡게 된 이유/어려움:** 협의 안 된 시안 + 2달 데드라인 + 기존/신규 동시
- **② 극복:** 기획 협의 병행 + Claude Code 컨텍스트 엔지니어링 + 검증 루프
- **③ 결과:** 2달 내 리뉴얼 완료 / ⬜지표

### 📝 문항 3 — 직무 역량 향상 노력·경험 (2개) — ✅ 확정
**경험 ① 에임메드 챗봇 빌더 시스템** (생성형 AI 서비스 인접)
- 수행기간: ⬜ / 역할: 풀스택 / 성과: ⬜ / 증빙: ⬜

**경험 ② 마크로젠 LIMS 해외법인(유럽·싱가포르) 독립운영** (풀스택 + 레거시 Java)
- 수행기간: ⬜ / 역할: 풀스택(FE+Java BE) / 성과: 독립운영 체계 완성 + ⬜지표 / 증빙: ⬜

> 🔀 **배치 변경 이력:** LIMS는 원래 문항2 후보였으나, KB 리뉴얼을 문항2로 올리며 LIMS를 문항3 경험②로 이동(프로젝트 중복 방지).

## 💡 아직 채워야 할 것 (본인 확인 필요)
1. **문항 2 소재 확정** — 위 (A)/(B)/(C) 중 "가장 포기하고 싶었지만 버틴" 것
2. **수치·증빙** — 챗봇빌더 도입 규모, 리뉴얼 후 지표(로딩속도/전환율/버그감소 등), 각 프로젝트 정확한 기간
3. **자소서 글자 수 제한** — 지원 폼의 문항별 제한(예: 1000자) 확인 → 분량 맞춤
