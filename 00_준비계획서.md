## 2. 직무·분야 공부 (Forward Deployed Engineer + AX)

### 2.1 Forward Deployed Engineer(FDE)란?
Palantir가 만든 모델. 핵심은 아래.
- **현장 밀착형 엔지니어**: 고객(사내 현업 부서) 팀에 직접 투입되어 함께 일함
- **"슬라이드가 아니라 코드를 배포한다(ship code, not slide decks)"** — 실제 프로덕션 솔루션을 빠르게 프로토타이핑·전달
- **End-to-end 오너십**: 스타트업 CTO처럼 문제 정의부터 개발·배포까지 소규모 팀으로 전담
- **비즈니스 + 기술 통역사**: 현업(비개발자)과 기술팀 사이 커뮤니케이션이 필수 역량
- **성과로 측정**: 투입 시간이 아니라 실제 업무 혁신(outcome)으로 평가

> **GC 맥락 해석:** GC녹십자의 FDE = R&D·제조·경영지원 등 사내 현업 부서에 들어가, 그들의 반복업무·데이터 문제를 **AI/에이전트 솔루션(Hey.GC 2.0 생태계)** 으로 직접 풀어내는 사람.

### 2.2 채용공고 원문 핵심 (2026 실제 공고)
**주요 업무**
- 생성형AI 서비스 개발 — **RAG 최적화, Agentic Workflow, Function Calling, Subagent, MCP**
- **AI 기반 개발도구(Claude Code, Cursor 등)** 활용한 프론트엔드·백엔드 개발
- **비즈니스 워크플로우 분석** → 최적 프로세스 설계

**자격요건 [필수]**
- **Python, JavaScript, Dart, Rust, Golang** 등으로 개발 가능
- **FastAPI, FastMCP, PydanticAI, LangChain, LangGraph, Enterprise LLM API** 활용 경험
- **ML/DL 이론 이해** 및 논리 기반 모델 설계 가능
- 사용자 입장에서 **"왜"**를 던지고 **구조적 해결**을 설계할 수 있는 분

**자격요건 [우대]**
- 인공지능·컴퓨터공학 전공
- 제약/바이오/헬스케어 산업 이해
- **사내 업무 프로세스에 AI를 이식해 생산성을 개선해 본 실무 경험** ← FDE 정체성 그 자체

### 2.3 "풀스택 AI 엔지니어" 기술 스택 → 내 역량 매핑
| 공고 요구 | 학습/증명 포인트 | 나의 현재 수준 (채워넣기) |
|---|---|---|
| **RAG 최적화** | 벡터DB, 임베딩, 청킹, 리랭킹, 검색품질 평가 | ⬜ |
| **Agentic Workflow / Subagent** | 에이전트 루프, 멀티스텝 툴사용, 서브에이전트 오케스트레이션 | ⬜ |
| **Function Calling** | 툴 콜링, 구조화 출력 | ⬜ |
| **MCP / FastMCP** | ⭐GC 핵심. MCP 서버·클라이언트, 사내 시스템 연결 | ⬜ |
| **FastAPI (백엔드)** | API 설계, 인증, DB | ⬜ |
| **LangChain / LangGraph** | 그래프 기반 에이전트 워크플로우 | ⬜ |
| **PydanticAI** | 타입 안전 LLM 앱 | ⬜ |
| **AI 개발도구 (Claude Code, Cursor)** | 실제 사용 경험 어필 | ⬜ |
| **프론트엔드** | React 등 챗/대시보드 UI | ⬜ |
| **ML/DL 이론** | 모델 설계 논리 | ⬜ |

> ⚠️ **"⬜ 나의 현재 수준" 칸을 반드시 채우기.** 이게 자소서 3번(직무 역량)과 1차 직무면접의 뼈대. 안 되는 항목은 프로젝트/학습으로 보완.

### 2.4 학습 방법 (부족한 것 위주로)
- [ ] **MCP + FastMCP**: 공식 문서 + 간단 MCP 서버 직접 제작 (프로젝트와 연결) ← 최우선
- [ ] **LangGraph or PydanticAI**: 에이전트 워크플로우 튜토리얼 1개 완주
- [ ] **RAG 최적화**: 청킹·리랭킹·평가까지 포함한 미니 파이프라인
- [ ] **Claude Code / Cursor** 실사용 → 자소서·면접에서 "AI 도구로 개발한다"를 자연스럽게
- [ ] "제약사에서 AI가 푸는 실제 문제" 사례 3개 (규제문서 검색, 문서요약, 실험데이터 정리 등)

## 4. 자소서 작성 계획

> **문항 요구 톤:** GC 자소서는 **구체적 경험 + 구조화(왜/어떻게/결과)** 를 명시적으로 요구. 두루뭉술 금지. **STAR 기법**(Situation-Task-Action-Result)으로 쓰면 정확히 맞음.

### ⭐ 4.0 (선행) 경험 발굴·정리 — 2·3번의 재료 만들기
자소서 2·3번은 **결국 "내 경험"이 재료**다. 글쓰기 전에 경험을 먼저 캐낸다. → 별도 파일 **`06_경험_인벤토리.md`**에 아래를 채운다.

**STEP 1. 경험 브레인스토밍 (양 먼저, 질은 나중)**
아래 카테고리별로 떠오르는 대로 다 적는다 (제목 + 1줄).
- [ ] 개발/AI/데이터 프로젝트 (학교 과제, 개인, 회사, 사이드)
- [ ] 공모전 / 해커톤 / 대외활동
- [ ] 연구 / 논문 / 졸업작품
- [ ] 실무 경험 (인턴, 직장, 외주) — 특히 **"AI로 업무 생산성 개선"** 류 (우대사항 직격)
- [ ] 학습 경험 (부트캠프, 강의, 자격증, 독학한 기술)

**STEP 2. 각 경험을 STAR로 한 줄 정리**
| 경험명 | 기간 | 내 역할 | 성과(수치/증빙) | 어려움/실패 | 증빙 링크 |
|---|---|---|---|---|---|

**STEP 3. 문항별 소재 배정**
- **2번용** (중도포기 위기 극복): 위기·좌절·끈기가 드러나는 것 1개 선택
- **3번용** (직무역량 향상, 2개↑): 공고 요구스택(RAG/Agent/MCP/풀스택)과 겹치는 것 우선, **객관적 증빙 가능**한 것으로 2개+
- 겹쳐도 됨. 단 2번과 3번에 **같은 경험을 그대로 재탕하진 않기** (각도 다르게)

> 📌 **오늘(7/7) 액션:** 아래 "내가 채워야 할 것"에 경험 후보를 최소 5개 적기부터 시작.
> → 이 부분은 **본인만 아는 정보**라, 내가 대신 못 씀. 후보를 주시면 어떤 걸 어느 문항에 쓸지, 어떻게 STAR로 풀지 같이 설계해 드림.

### 문항 1. 회사 선택 기준 + GC녹십자 지원 이유
**구성 전략:**
1. **나만의 회사 선택 기준** 2가지 (예: ① 기술로 실제 문제를 푸는 조직 ② 사회적 가치가 명확한 산업)
2. 그 기준에 GC녹십자가 **어떻게 부합하는지** 연결
   - AX 전사 추진 + Hey.GC 2.0(Agentic AI/MCP) → "기술로 진짜 업무를 바꾸는 회사"
   - 혈액제제·백신 → "생명을 다루는 명확한 사회적 가치"
3. FDE로서 **내가 무엇을 기여할지** 한 문장으로 마무리
- **소재 준비:** `01_회사분석.md`에서 최신 AX 근거 인용

### 문항 2. 중도 포기할 수 있었지만 완수한 경험 (①~③ 구조 필수)
**요구 구조 그대로 목차화:**
- **① 맡게 된 이유 / 어려웠던 이유**
- **② 포기하고 싶던 순간 / 극복 노력 / 느낀 점**
- **③ 수행 결과 / 이를 통해 느낀 바** (구체적으로)
- **소재 후보(택1, 미리 정하기):** 난이도 높았던 프로젝트/졸업과제/업무/공모전 중 "끝까지 붙잡아 완성한" 것. 위 프로젝트도 후보.
- **작성 팁:** 감정(포기하고 싶던 순간)을 솔직하게 + 극복 행동을 **구체적 액션**으로 + 결과를 **수치/증빙**으로.

### 문항 3. 직무 역량 향상 노력·경험 (**2개 이상**, 증빙 가능)
**필수 포함 요소:** 수행기간 / 지원자 역할 / 성과 / (실패 시) 실패로 느낀 점
- **경험 A:** 위 3장 프로젝트 (MCP·RAG·Agentic) → 공고 필수스택(FastMCP/LangGraph 등)과 직접 연결
- **경험 B:** 기존 실무·연구·학습 경험 중 AI/개발 관련 (데이터 분석, 백엔드, 논문/공모전 등)
- **⭐ 우대사항 저격:** 가능하면 **"사내/실무 업무 프로세스에 AI를 이식해 생산성을 개선한 경험"** 을 한 축으로. 이게 공고 우대사항이자 FDE 정체성.
